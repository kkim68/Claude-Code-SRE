## Coverage Gap Summary

Three server-side files were not scanned and warrant follow-up:

1. **`server/index.ts`** — Priority high. **Why skipped:** The file is 3.5 MB because it embeds a large block of Express.js documentation as inline comments. The sub-agent treated it as a documentation or boilerplate file rather than runtime application code and did not open it. 
2. **`server/routes/reviews.ts`** — Priority medium. **Why skipped:** The sub-agent stopped after confirming high-severity vulnerabilities in `auth.ts` and did not continue to the remaining route files. Likely shares the same SQL injection pattern.
3. **`server/routes/watchlist.ts`** — Priority medium. **Why skipped:** Same reason as `reviews.ts` — the agent concluded its scan before reaching this file. Same SQL injection risk applies.
