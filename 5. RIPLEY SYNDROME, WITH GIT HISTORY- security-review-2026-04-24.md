## Vuln 1 (HIGH): Arbitrary File Read via Vite Dev Server Path Traversal

**File:** `package-lock.json` · `vite.config.ts`
**Severity:** High
**Category:** `path_traversal` / `arbitrary_file_read`
**Confidence:** 0.95

### CVEs

| CVE | Affected Range (4.x) | CVSS |
|---|---|---|
| CVE-2024-23331 | Vite < 4.5.2 | 7.4 (High) |
| CVE-2025-30208 | Vite < 4.5.10 | High |

### Confirmed Version

```
"node_modules/vite": {
  "version": "4.0.4"   ← locked in package-lock.json
}
```

### Description

Vite 4.0.4 ships with two related vulnerabilities in its development server:

- **CVE-2024-23331:** `server.fs.deny` allowlist can be bypassed via crafted URL paths, allowing requests to escape the project root and read arbitrary filesystem files.
- **CVE-2025-30208:** A double-question-mark query suffix (`?raw??`) bypasses `server.fs.deny` entirely. A request to `/@fs/<absolute-path>/.env?raw??` returns the file contents even when that path should be restricted.

### Exploit Scenario

The project contains a `.env` file at the repository root (confirmed present) and the `run` script starts the Vite dev server on port `5173`. An attacker with access to that port sends:

```http
GET /@fs/Users/xsaprvd/Documents/GitHub/Claude-Code-SRE/.env?raw??  HTTP/1.1
Host: localhost:5173
```

The dev server bypasses `server.fs.deny` and returns the raw `.env` contents — including any `VITE_TMDB_API_KEY` or other secrets. The same path works for any file readable by the server process (e.g., `~/.ssh/id_rsa`, `/etc/passwd`).

### Recommendation

Upgrade Vite to `>= 4.5.10` in `package.json`:

```json
"vite": "^4.5.10"
```

Then run:

```sh
npm install
```

Additionally, never run `vite dev` with `server.host: true` or the `--host` flag on an untrusted or shared network.
